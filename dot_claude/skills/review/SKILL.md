# コードレビュー

## 手順

以下のBashコマンドでコンテキストを収集してからレビューを行う:

```bash
# 現在のブランチ
git branch --show-current

# このブランチのコミット（mainにないもの）
git log main..HEAD --oneline

# 変更ファイル一覧
git diff --stat main

# 全差分
git diff main
```

## レビュー観点

1. **サマリー** - 変更の概要と目的
2. **コード品質** - スタイル一貫性・可読性・保守性
3. **潜在的バグ** - ロジックエラー・エッジケース・エラーハンドリング
4. **セキュリティ** - 脆弱性・機密データの露出
5. **パフォーマンス** - パフォーマンスへの影響
6. **改善提案** - より良い実装のための提案

## アウトプット方針

- 簡潔かつ実行可能な指摘に絞る
- 重要度の高い問題を優先する
- 指摘は具体的なファイル・行番号を示す

### レビュー粒度

**必ず指摘する:**
- バグ・ロジックエラー（動作に影響するもの）
- セキュリティ脆弱性
- 計測済みまたは明らかなパフォーマンス問題

**指摘しない:**
- Nitpick（変数命名の好み・スタイルの微差）
- 根拠のない仮説レベルの懸念（「もしかしたら問題になるかも」）
- 既存コードと無関係な改善提案

**例外的に触れる:**
- Nitpick が diff 全体に目立つ場合のみ「スタイルガイド準拠を検討」と1行で言及する

## ファイル保存

レビュー完了後、以下のパスに結果を保存する:

- ディレクトリ: `~/Documents/Development/claude-output/reviews/`
- ファイル名: `YYYY-MM-DD_HH-MM_{ブランチ名}.md`
- 内容: レビュー結果全文（Markdown）
