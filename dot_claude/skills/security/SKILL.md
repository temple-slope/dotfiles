# セキュリティレビュー

## 手順

以下のBashコマンドでコンテキストを収集してから分析を行う:

```bash
# 現在のブランチ
git branch --show-current

# 変更ファイル一覧
git diff --name-only main

# 全差分
git diff main

# シークレット候補をスキャン
git diff main | grep -iE '(api[_-]?key|api[_-]?secret|access[_-]?token|secret[_-]?key|password\s*=|private[_-]?key|-----BEGIN|AWS_|GITHUB_TOKEN|OPENAI_API)'

# Base64 長文字列（潜在的埋め込みシークレット）
git diff main | grep -oE '[A-Za-z0-9+/]{40,}={0,2}' | head -20
```

## チェック項目

### 1. シークレット・認証情報

**検査方法**: 上記スキャンコマンドのヒット行を確認する

- ハードコードされたAPIキー・トークン・パスワード
- 秘密鍵・証明書
- 認証情報を含む接続文字列
- 機密設定値（1Password参照の露出含む）

### 2. インジェクション脆弱性

**検査方法**: ユーザー入力が SQL/シェル/eval に直接渡されていないか差分を目視確認

- SQLインジェクション
- コマンドインジェクション
- XSS（クロスサイトスクリプティング）
- コードインジェクション・eval使用

### 3. 認証・認可

**検査方法**: 認証チェックを削除・バイパスするコード変更がないか確認

- アクセス制御の不備
- 認証チェックの欠落
- 安全でないセッション管理
- 権限昇格リスク

### 4. データ露出

**検査方法**: ログ出力・レスポンスボディに PII やシークレットが含まれないか確認

- ログへの機密データ出力
- 暗号化されていない機密データ
- APIレスポンスの過剰なデータ露出
- PII（個人識別情報）の不適切な扱い

### 5. 入力バリデーション

**検査方法**: 外部入力を受け取る関数でバリデーションが追加・維持されているか確認

- バリデーションの欠落・不十分
- パストラバーサル脆弱性
- 安全でないデシリアライゼーション
- ファイルアップロードの脆弱性

### 6. 依存関係・設定

**検査方法**: 追加・更新されたパッケージと設定変更を確認

- 既知の脆弱な依存関係
- 安全でないデフォルト設定
- セキュリティヘッダーの欠落
- 過度に許可的なCORS設定

## 深刻度判定基準

| 深刻度 | 条件 | 対応 |
|---|---|---|
| Critical | シークレット・秘密鍵が diff に含まれる | コミット即停止。既コミット済みの場合は `git filter-repo` をユーザーに依頼 |
| High | インジェクション可能なコードパス・認証バイパス | 修正必須。修正前にプッシュしない |
| Medium | PII のログ出力・過剰なデータ露出 | 修正推奨。ユーザーに判断を委ねる |
| Low | スタイル上の懸念・将来リスク | 指摘のみ。ブロックしない |

## アウトプット形式

問題が見つかった場合、各項目を以下の形式で報告:

- **深刻度**: Critical / High / Medium / Low
- **場所**: ファイルと行番号
- **問題**: 脆弱性の説明
- **推奨対応**: 修正方法

### 出力例（問題ありケース）

```
## セキュリティレビュー結果

### Critical: ハードコードされた APIキー

- **場所**: `src/config.ts:12`
- **問題**: `OPENAI_API_KEY = "sk-proj-..."` が直接コードに含まれている
- **推奨対応**: 環境変数 (`process.env.OPENAI_API_KEY`) に移行し、値を削除して `git filter-repo` で履歴を浄化する

---

**判定: 不合格（Critical 1件）。修正完了後に再レビューを実施してください。**
```

問題が見つからない場合は、変更が安全であることを確認して報告する。

```
## セキュリティレビュー結果

チェック項目（1〜6）に問題は見つかりませんでした。

**判定: 合格**
```

## ファイル保存

レビュー完了後、以下のパスに結果を保存する:

- ディレクトリ: `~/Documents/Development/claude-output/security/`
- ファイル名: `YYYY-MM-DD_HH-MM_{ブランチ名}.md`
- 内容: セキュリティレビュー結果全文（Markdown）
